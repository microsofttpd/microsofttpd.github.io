<p><span style="text-decoration:underline;">Procedure:</span><p>In order to create an integration with AX and CRM custom entity:</p><p>Step 1:1 Create a custom new EDT(Extended data type), and create new table while creating the table add DAXIntegrationID as a field and extend ConnIntegrationKey also click the modifiedDate property as Yes. Add insert method and add event handler extend Dynamicsconnector class.</p><p>Step 1:2 Create form and add the created table as DS. Also create a query and in query change the update property as yes in AX 2012.</p><p>Step 1:3 Create document service in AIF(Application Integration Framework) [Path: Development workspace &ndash;&gt; tools &ndash;&gt; AIF &ndash;&gt; -&gt; Create document service &ndash;&gt; choose created query]</p><p>Step 1:4 Create an Inbound Port for the AIF Service</p><p>Step 1:5 Click on Service operations and select the read, write and update for the created service class.</p><p>Step 1:6 And Activate AIF inbound port.</p><p>Step 1:7 Go to connector map click on the configuration settings and configure the services for Axapta</p><p>Step 1:8 Once the configuration settings(Adapter configuration) have been clicked, click on refresh services,</p><p>Step 1:9 It will display the available document services in AX AIF, enable that check mark and click on configure button</p><p>Step 1:10 Go to the physical path of the connector adapter for Axapta [C:Program Files (x86)Microsoft DynamicsMicrosoft Dynamics AdapterAdaptersMicrosoft.Dynamics.Integration.Adapters.Ax2012ObjectConfigdax2012_8101] and change the Queryname in the &lt;Root &gt;tag with the AXdqueryName(once AIF document service has been created AXdQueryName) and save it.</p><p>Step 1:11 In CRM 2011 create new solutions [Path: Settings &ndash;&gt;
